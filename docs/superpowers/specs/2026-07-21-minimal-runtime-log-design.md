# Yoinks 最小运行日志设计

## 目标

在“调试模式”默认关闭时，仍保留可供智能体诊断的、经脱敏的关键运行流程日志；日志独立保存并严格限制为 128 KB。开启调试模式后，将这部分日志安全并入现有主日志，再恢复完整结构化调试记录。

## 范围

- 新增独立文件 `logs/minimal.jsonl`，保持与现有日志一致的 JSONL 事件格式。
- 调试关闭时，不写 `latest.jsonl` 或任务历史日志；只写最小日志。
- 最小日志记录关键流程节点及失败：链接提交/校验、媒体探测、下载、合并、校验、保存、历史记录/清理、登录重试。
- 最小日志最大 128 × 1024 字节；超过后仅保留尾部完整 JSONL 事件。
- 调试开启时：解析最小日志、再次脱敏、按时间合并到 `latest.jsonl`、清空最小日志，然后记录 `debug-mode.enabled`。
- 新增受控读取接口，返回解析且再次脱敏的事件，供智能体使用；调用方不读取原始 JSONL 文件。

## 非目标

- 不在普通用户界面增加最小日志的查看、复制或目录入口。
- 不改变现有主日志 512 KB、任务历史 4 MB、主日志筛选或历史归档规则。
- 不将关闭调试时的最小日志写入任务历史文件。

## 安全与容错

- 沿用现有 URL、Cookie、令牌、密码和授权字段脱敏；读取时再次执行清洗。
- 损坏或不完整的 JSONL 行跳过，不阻断下载或调试模式切换。
- 文件读写和合并失败静默降级，不影响主功能；成功合并后才清空最小日志。

## 验证

- 服务级验证：调试关闭的最小写入、关键事件筛选、128 KB 截断、开启合并/清空、再次脱敏与损坏行容错。
- TypeScript 项目诊断。
- `scripting-ts project "Yoinks"` 启动回归；因前台 UI 生命周期导致的超时须如实报告。
